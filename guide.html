How to Put Your Portfolio Online

Follow these easy steps — no tech skills needed.

Back to Generator
What you need: The ZIP file you just downloaded + a free GitHub account. That's it. Takes ~5 minutes
1

Make a free GitHub account

Already have one? Skip to Step 2.

  • Go to github.com/signup
  • Enter your email, pick a password, and choose a username
  • Finish the sign-up and verify your email
Remember your username! Your portfolio link will be: yourusername.github.io/portfolio
2

Create a new project (repository)

Think of this as a folder where your website files will live.

  • On GitHub, click the + button (top-right corner)
  • Choose "New repository"
  • Type portfolio as the name
  • Make sure "Public" is selected
  • Check the box that says "Add a README file"
  • Click the green "Create repository" button
3

Unzip and upload your files

  • Find the ZIP file you downloaded and right-click → Extract All
  • Open the extracted folder — you'll see index.html and style.css
  • Go back to your GitHub repository page
  • Click "Add file" then "Upload files"
  • Drag both files into the upload area
  • Scroll down and click "Commit changes"
Make sure you upload the files directly — not inside a subfolder. The files should appear right next to the README file.
4

Turn on GitHub Pages

This is what makes your files show up as a real website.

  • On your repository page, click "Settings" (the gear icon tab)
  • On the left side, click "Pages"
  • Under "Source", pick "Deploy from a branch"
  • Select the "main" branch and "/ (root)"
  • Click "Save"
5

Wait 2 minutes, then check your site

  • Wait about 1–3 minutes for GitHub to set everything up
  • Refresh the Pages settings page — you'll see a green box with your link
  • Your portfolio is live at: yourusername.github.io/portfolio
You're done! Share your link on your resume, social media, email signature — anywhere you want people to see your work.

Common Questions

How do I update my portfolio later?

Just go back to the Auto Portfolio Generator, make your changes, download a new ZIP, and upload the new files to the same repository. They'll replace the old ones automatically.

My site shows a 404 error?

Make sure index.html is in the root of your repository (not inside a folder). Also double-check that GitHub Pages is turned on in Settings → Pages.

Can I use my own domain name?

Yes! In your repository's Pages settings, there's a "Custom domain" field. Enter your domain there and follow the DNS instructions GitHub gives you.

Does this cost anything?

Nope. GitHub Pages is 100% free for public repositories.
