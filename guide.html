Deploy Your Portfolio to GitHub Pages

A step-by-step guide to publish your portfolio website for free.

← Back to Portfolio Generator
What you'll need: Your downloaded portfolio ZIP file and a free GitHub account. This guide will walk you through every step — no coding experience required. Your portfolio will be live at yourusername.github.io/portfolio within minutes.

1 Create a GitHub Account

If you already have a GitHub account, skip to Step 2.

  1. Go to github.com
  2. Click "Sign up" in the top right corner
  3. Enter your email, create a password, and choose a username
  4. Complete the verification and confirm your email
Screenshot: GitHub sign-up page

2 Create a New Repository

A repository is like a folder that will hold your portfolio files.

  1. Click the "+" icon in the top-right corner of GitHub
  2. Select "New repository"
  3. Name it portfolio (or any name you like)
  4. Set it to Public
  5. Check "Add a README file"
  6. Click "Create repository"
Screenshot: Create new repository form
Tip: If you name the repository yourusername.github.io, your site will be available at yourusername.github.io directly (without the /portfolio path).

3 Upload Your Portfolio Files

Now upload the files from your downloaded ZIP.

  1. Unzip the downloaded portfolio ZIP file on your computer
  2. In your GitHub repository page, click "Add file" → "Upload files"
  3. Drag and drop both files: index.html and style.css
  4. Scroll down and click "Commit changes"
Screenshot: Upload files to GitHub
Important: Make sure you upload the files to the root of the repository, not inside a subfolder. The index.html must be at the top level.

4 Enable GitHub Pages

This tells GitHub to serve your files as a website.

  1. Go to your repository page
  2. Click "Settings" (gear icon tab)
  3. In the left sidebar, click "Pages"
  4. Under "Source", select "Deploy from a branch"
  5. Select the "main" branch and "/ (root)" folder
  6. Click "Save"
Screenshot: GitHub Pages settings

5 Wait for Deployment

GitHub will now build and deploy your site. This usually takes 1-3 minutes.

  1. Stay on the Pages settings page
  2. Refresh the page after a minute or two
  3. You'll see a green banner with your site URL
Screenshot: Site deployed banner with URL

6 Visit Your Live Portfolio!

Your portfolio is now live on the internet.

  1. Your URL will be: https://yourusername.github.io/portfolio
  2. Share this link on your resume, LinkedIn, email signature, or social media
  3. To update your portfolio later, just upload new files to the same repository
Congratulations! Your portfolio is now live for the world to see. You can share this URL anywhere — it's yours forever (as long as the repository exists).

1 Install GitHub Desktop

GitHub Desktop is a free app that makes working with GitHub easier.

  1. Go to desktop.github.com
  2. Download and install the app for your operating system
  3. Open GitHub Desktop and sign in with your GitHub account
Screenshot: GitHub Desktop download page

2 Create a New Repository

  1. In GitHub Desktop, click "File" → "New Repository"
  2. Name it portfolio
  3. Choose a local path where you want to store the files
  4. Check "Initialize this repository with a README"
  5. Click "Create Repository"
Screenshot: GitHub Desktop — Create repository dialog

3 Add Your Portfolio Files

  1. Unzip your downloaded portfolio ZIP file
  2. Copy index.html and style.css into the repository folder on your computer
  3. GitHub Desktop will automatically detect the new files
Screenshot: GitHub Desktop showing changed files

4 Commit & Push

  1. In the bottom-left, type a commit message: Add portfolio files
  2. Click "Commit to main"
  3. Click "Publish repository" (or "Push origin" if already published)
  4. Make sure "Keep this code private" is unchecked
Screenshot: GitHub Desktop — Commit and push

5 Enable GitHub Pages

Now switch to the browser to enable hosting.

  1. In GitHub Desktop, click "Repository" → "View on GitHub"
  2. On the GitHub website, go to "Settings" → "Pages"
  3. Under Source, select "main" branch and "/ (root)"
  4. Click "Save"
  5. Wait 1-3 minutes, then visit your live URL
Screenshot: GitHub Pages settings
Your site is live! Visit https://yourusername.github.io/portfolio to see your portfolio. To make updates, just edit files locally, commit, and push again.

Frequently Asked Questions

Q: How do I update my portfolio later?

Simply regenerate your portfolio with the Auto Portfolio Generator and re-upload the new files to your repository. The changes will go live automatically within a few minutes.

Q: Can I use a custom domain?

Yes! In your repository's Pages settings, you'll find a "Custom domain" section. Enter your domain name there, and follow GitHub's instructions to update your DNS records.

Q: My site shows a 404 error. What do I do?

Make sure your index.html file is in the root of the repository (not inside a folder). Also verify that GitHub Pages is enabled and set to the correct branch.

Q: Can I make my repository private?

GitHub Pages works with public repositories on free accounts. If you need a private repository, you'll need GitHub Pro.
